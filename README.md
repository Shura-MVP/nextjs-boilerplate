# شــورى — Shura

> منظومة المداولة الاستراتيجية متعددة الوكلاء

---

## نظرة عامة

شــورى منظومة مُحكَمة لصنع القرار الاستراتيجي، تُجري مداولة منهجية بين مجموعة من الوكلاء المتخصصين، يَزِن فيها كلٌّ منهم السؤال من زاوية تخصصه، قبل أن يُؤلَّف الموجز الاستراتيجي النهائي بصيغة قابلة للتنفيذ.

---

## المبادئ الجوهرية

| المبدأ | الوصف |
|--------|-------|
| **الاستشهاد قبل الادعاء** | كل ادعاء جوهري يحمل وسماً يربطه بمصدره، أو يُفصح صراحةً عن طبيعته (تعليل منطقي / فجوة معرفية). |
| **تعدد الزوايا** | لا تُعتمد زاوية واحدة. السؤال يُستعرض من جوانبه الموضوعية والعدائية والمعرفية في آنٍ معاً. |
| **الموازنة المُحكَمة** | الموجز النهائي يوازن بين الأدلة والآراء المتعارضة، ولا يرجّح إلا بدليل. |
| **الإفصاح عن الفجوات** | ما لا تعرفه المنظومة يُذكر صراحةً، لا يُخفى ولا يُغطَّى. |

---

## البنية التقنية

- **الإطار:** Next.js 15 (App Router) + React 19
- **اللغة:** TypeScript
- **التنسيق:** Tailwind CSS
- **الأنميشن:** Framer Motion
- **الثيم:** next-themes (وضع فاتح / داكن)
- **الذكاء:** Anthropic Claude API
